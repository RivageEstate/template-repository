# Security Policy

脆弱性の疑いがある問題は、公開 Issue ではなく非公開チャネルで報告してください。

## 報告方法

- GitHub Security Advisories の Private vulnerability reporting を優先
- 利用できない場合は、リポジトリ管理者へ直接連絡

## 初動SLA目安

- 受付連絡: 3営業日以内
- トリアージ結果共有: 7営業日以内
- 修正方針または暫定対策の提示: 14営業日以内

## 公開方針

- 修正前に詳細情報を公開しない
- 修正後に影響範囲、回避策、修正バージョンを公開する
- 報告者クレジットは希望を確認したうえで掲載する
